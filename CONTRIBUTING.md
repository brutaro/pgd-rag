# Guia de Contribuição

Obrigado por considerar contribuir para o PGD RAG! Este documento fornece diretrizes para contribuir.

## Como Contribuir

### Reportar Bugs

Se você encontrou um bug:

1. Verifique se já não existe uma issue aberta sobre o problema
2. Abra uma nova issue descrevendo:
   - Descrição clara do bug
   - Passos para reproduzir
   - Comportamento esperado vs. comportamento observado
   - Ambiente (Python version, OS, etc.)

### Sugerir Melhorias

1. Abra uma issue com a tag `enhancement`
2. Descreva claramente a melhoria proposta
3. Explique o valor/benefício da melhoria

### Pull Requests

1. Fork o repositório
2. Crie uma branch (`git checkout -b feature/nova-feature`)
3. Faça commit das mudanças (`git commit -m 'Adiciona nova feature'`)
4. Push para a branch (`git push origin feature/nova-feature`)
5. Abra um Pull Request

### Padrões de Código

- Use Python 3.10+
- Siga PEP 8 (formatação)
- Adicione comentários para código complexo
- Escreva docstrings para funções públicas
- Teste suas mudanças antes de fazer PR

### Commits

Use mensagens de commit descritivas:
- `feat: Adiciona nova funcionalidade`
- `fix: Corrige bug na detecção`
- `docs: Atualiza README`
- `refactor: Refatora função X`
- `test: Adiciona testes para Y`
